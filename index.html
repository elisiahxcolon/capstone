Implementation

Network Scanner (NS)
The Network Scanner currently meets the original proposed functionality. It:

  • Discovers the requesting device's hostname and local IP address.
  • Scans open and closed ports on a target IP.
  • Discovers devices on the local network (ARP-based fallback).
  • Measures and reports current ping and approximate download speed.
  • Provides a simple Pygame-based interface for visualization and interaction.

The NS was developed in Python using PyCharm. While the original goal included integrating tools such as Nmap and Wireshark, those tools were not fully integrated due to time and compatibility constraints. Instead, the implementation relies on standard and third-party Python libraries for network queries and measurements, for example:

  • requests
  • socket
  • subprocess
  • urllib.request
  • os, sys

Key NS functions

  • get_public_ip() — retrieves public IP via the ipify API.
  • get_local_ip() — retrieves the machine's local IP assigned by the router.
  • get_isp(ip) — looks up ISP information (e.g., via ipinfo.io).
  • scan_ports(ip, ports, timeout=0.3) — checks a list of ports and reports open/closed status.
  • arp_scan_fallback() — uses the system ARP table to discover local devices (IP + MAC).
  • measure_download_speed() and draw_speed_meter(speed) — measure download throughput and display a Pygame speed meter.

Simulated Router (SR) & Anomaly Detection
The Simulated Router creates a dynamic, testable network and integrates with a simple anomaly detection system. It generates devices with randomized IPs and port configurations and exposes that data to the NS for evaluation.

SR & ML components

  • SimulatedRouter.generate_devices(num_devices) — generates devices with IPs and random open-port sets.
  • SimulatedRouter.expose_data_for_scanner() — returns device configuration data in a structured format for the scanner.
  • AnomalyDetector.prepare_features(devices) — converts devices to numerical feature vectors (e.g., last IP octet, number of open ports).
  • AnomalyDetector.train_and_detect(devices) — uses K‑Means clustering to find outliers; smallest clusters are flagged as anomalous.

Example: if policy expects ≤ 3 open ports per device, a device with 10+ open ports will likely be clustered as an outlier and flagged for review.

Planned improvements / future work

  • Implement a web-based front end instead of Pygame for richer UI and easier sharing.
  • Integrate tools like Nmap and Wireshark by running them inside controlled virtual machines or isolated environments.
  • Use virtual machines to safely test private-network behaviors and legality-sensitive experiments (e.g., simulated DDOS testing).
  • Expand SR realism: add hostname, simulated OS fingerprints, richer metadata, and policy-driven configuration rules (banned ports, whitelists).
  • Build a hybrid detection engine: ML clustering + rule-based policy engine for stronger anomaly detection.
  • Expose a REST API so the Network Scanner can query simulated networks dynamically.
  • Visualize detection output in real time on a web dashboard and add more advanced traffic simulation (Scapy) where appropriate.

The long-term goal is to evolve this into a lightweight intrusion-detection / simulation environment that balances practical tooling with safe, repeatable testing workflows.

Former Implementation

Network Scanner Overview

Development & Environment:
The Network Scanner (NS) was developed entirely in Python and tested on a home router to simulate real-world network environments.
A simple PyGame interface was used to provide a graphical UI for displaying network information.
The implementation was functional but limited in versatility, focusing mainly on local discovery and basic scanning capabilities.

Core Functionalities:
- get_public_ip() – Retrieves the public IP address of the machine using the ipify API.
- get_local_ip() – Detects the local IP assigned by the router.
- scan_ports(ip, ports) – Scans specified ports on a device and reports open or closed status.
- arp_scan_fallback() – Uses the system’s ARP table to detect local network devices.
- measure_download_speed() – Estimates download speed by transferring sample data.

Tools & Libraries:
All code was written and tested in the PyCharm IDE using imported Python libraries such as socket, subprocess, and requests.
These tools worked as lightweight substitutes for heavier network tools like Nmap or Wireshark, which were planned but not fully integrated.

Future Goals:
- Transition the UI from PyGame to a web-based front end for more flexibility and scalability.
- Improve cross-compatibility and integrate more advanced APIs for real-time network monitoring.

Network Familiarity

Windows Server: A server operating system developed by Microsoft that manages network resources, users, and security. It provides the environment to test and experiment with the Network Scanner and simulate client-server interactions.

DHCP (Dynamic Host Configuration Protocol): A network protocol that automatically assigns IP addresses to devices on a network. It ensures devices, including virtual machines, receive IP addresses dynamically, which the Network Scanner can detect and analyze.

DHCP Scope: A range of IP addresses that a DHCP server can assign to devices. This allows controlled IP distribution across the network.

IP Address: A unique identifier assigned to each device on a network. IP addresses allow devices to communicate with each other and are essential for the Network Scanner to locate devices.

ISO: A disk image format used to distribute operating systems. The Windows Server ISO will be used to deploy the server in a virtual machine (VMware) for testing.

Firewall: A network security system that monitors and controls incoming and outgoing network traffic. It ensures only authorized traffic is allowed, helping to simulate a secure environment for the Network Scanner.

Relevance to the Project: This setup allows safe experimentation with the Network Scanner, understanding IP assignments, device detection, and client-server relationships, all within a controlled virtual environment.
